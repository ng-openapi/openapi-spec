How these files are produced

REFERENCE.yaml at the repo root is the single source of truth. It defines one API plus per-version rendering rules; the three files under specs/ are generated from it.

  1. Edit REFERENCE.yaml to add or change endpoints, schemas, security, etc.
  2. Regenerate by asking Claude: "regenerate specs from REFERENCE.yaml".
  3. Validate — CI runs Redocly and Spectral against all three rendered specs.
